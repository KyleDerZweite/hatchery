# Security Policy

## Supported Version

Only the current `main` branch is supported.

## Reporting

Use GitHub private vulnerability reporting for vulnerabilities that could expose credentials, allow unauthorized access, or materially affect system integrity.

Do not open public issues for:

1. Credential leakage
2. Authentication or authorization bypass
3. Remote code execution
4. Panel API key handling flaws
5. Database access or encryption weaknesses

If private vulnerability reporting is unavailable for any reason, open a GitHub security advisory draft instead of a public issue.

## Response Expectations

This is a pre-1.0 MVP repository. Fixes are prioritized by exploitability, operational impact, and the amount of user data or infrastructure exposed.
